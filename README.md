# Snake Game in RISC-V Assembly (Ripes)
The game supports real-time snake movement, D-pad input handling, snake body growth, apple generation, rendering on an LED matrix display, and boundary reset logic.

A Snake Game implemented completely in RISC-V Assembly using the Ripes RISC-V ISA Simulator and memory-mapped I/O.

This project was developed as part of a project to demonstrate low-level programming concepts such as:

* Memory-mapped I/O
* Register manipulation
* Array-based body management
* Real-time input handling
* Game loops in assembly language
* LED matrix rendering

The game supports:

* Snake movement using D-Pad input
* Snake body growth
* Apple consumption detection
* Boundary collision reset
* LED matrix graphics rendering

# Important Setup Instructions

Before running the program, the memory-mapped I/O base addresses in the code **must match the addresses generated by your Ripes setup**.

The addresses used in this project are:

```assembly
.equ LED_BASE,    0xf0000000
.equ D_PAD_UP,    0xf0000dac
.equ D_PAD_DOWN,  0xf0000db0
.equ D_PAD_LEFT,  0xf0000db4
.equ D_PAD_RIGHT, 0xf0000db8
```

These values may differ depending on your Ripes version or connected peripherals.

## To Verify the Correct Addresses

1. Open Ripes
2. Add the LED Matrix peripheral
3. Add the D-Pad peripheral
4. Check the generated MMIO base addresses
5. Replace the `.equ` values in the source code if necessary
These are the values you need to correct.
<img width="2301" height="1295" alt="image" src="https://github.com/user-attachments/assets/c402dec8-cc83-4da6-a081-d4de74cd0e11" />
If the addresses are incorrect:

* The snake will not render properly
* Input controls may not work

# Game Logic

The game works using a continuous loop structure:

1. Read D-Pad input
2. Update snake direction
3. Move snake head
4. Shift snake body arrays
5. Detect apple collision
6. Increase snake length
7. Render snake and apple
8. Check boundary collision
9. Repeat loop

The snake body is managed using two arrays:

* `snake_x[]`
* `snake_y[]`

Each array stores the coordinates of body segments.
